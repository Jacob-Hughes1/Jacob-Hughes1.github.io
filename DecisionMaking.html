AI Soldier Decision Making Simulation

Unity Engine - Solo Project

Overview

This is a simulation that showcases an AI System I created using a Markov System to make decisions based on the agents current situation.
Made as part of my Game Artificial Intelligence Module during my second year at the University of Staffordshire.

KEY SYSTEM FEATURES

AI Behaviour States

The system has multiple states, which it can exist in all at once due to the use of the Markov System.
States include: Shooting, Fleeing, ItemPickup and Wander.
Each state affects the behaviour of the agent at different levels dependant on it's 'activation amount'.

Item Priority

Whenever the AI agent detects an item in the level, it priorities which item it wants to go for based off of their current health and ammo values.
The agent will take into account its desire for the item and prioritise the direction and speed it should be moving based off of nearby enemies.

Smart Enemy Avoidance

To make the AI agent feel intelligent, it uses a radius around itself to detect nearby enemies. It then calculates that enemy velocity and flees from the area where it thinks the enemy will make contact, it also flees from the area of the enemy itself.
If there are multiple enemies within the agents detection radius, it will take an average of all of their positions and attempt to avoid that, this creates the effect where the agent is weaving between the zombies to get away.

Zombie Behaviour

To improve the feel of the simulation, zombie behaviours were also tweaked to create 'mindless' feeling behaviour.
Zombies use a simple finite state machine, which causes them to only be in one state at a time. The three main zombie states are wander, pursue and seek. The combination of these states create the effect that the zombie becomes locked onto the agent and then charges mindlessly at them.
